## 🧠 Concept

SMAY is not a full sampler workstation.

It is:
- simple
- direct
- performance-focused

Each **slot = one sample**  
Each **song = a collection of slots**  
Each **set = a live setup**

No effects. No menus. Just sound.

---

## 🎛 Slot Features

Each slot supports:

- Key range
- Velocity range
- Velocity depth (positive + negative)
- Attack / Decay / Sustain / Release
- Loop start / end
- Zero-crossing snap
- Transpose
- Pitch bend range
- MIDI delay
- Vibrato (with modwheel support)

---

## 🔁 Looping

Looping is sample-based and includes:

- Manual loop points (in seconds)
- Automatic snap to nearest zero crossing
- Works well for sustained sounds (strings, pads, etc.)

Note: For perfect seamless loops, good source samples still matter.

---

## 🎼 Backing Tracks

Each song can include a backing track:

- Drag & drop audio file
- Independent volume control
- Play / pause / stop
- Stored inside `.smayset`

---

## 📦 Project Format

SMAY uses a custom `.smayset` format (zip-based):
project.json
samples/
backingtracks/


This makes sets:
- portable
- shareable
- self-contained

---

## 🚀 Getting Started

1. Unzip to a folder
2. Run smay.html
3. Add a song
4. Add slots
5. Drag samples into slots
6. Play via MIDI or computer keyboard

---

## 🎹 Controls

- MIDI input supported
- Computer keyboard supported (ZXCVB → piano)
- Pitch bend via keyboard arrows (optional)

---

## 🎯 Philosophy

SMAY is built around a simple idea:

> You should be able to build and play an instrument in minutes.

Not hours. Not menus. Not presets.

Just samples, MIDI, and you.
